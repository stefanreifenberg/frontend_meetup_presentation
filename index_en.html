The Essentials

What makes a visualization truly effective?

Before we dive into code: what actually makes a good dataviz?

Data Visualization

is the graphical representation of information and data.

transforms data values into visual, quantifiable forms.

helps to recognize, discover, explain and decide.

is on one hand an art and on the other hand a science.

This is the definition of data visualization. It is about representing information visually in order to recognize patterns and relationships. It is a tool for communication, analysis and storytelling.

Visual Encoding

Data values → visual channels (Cleveland & McGill, 1984)

Position
most precise ✓✓✓
Length
very precise ✓✓
Area
good ✓
Color (Hue)
for categories
Angle
imprecise ✗
Shape
for categories
This explains why bar charts are so powerful — position on a shared axis is the most precise channel. And why pie charts are problematic — comparing angles is hard.

The Craft

The craft behind great charts

Technically we can build almost anything. How do you do it well?

The Core Question

"So what?"

Every chart needs a clear message.
If you can't state it in one sentence,
the chart isn't done yet.

This is the most important question. What should the user take away? The title should convey the message, not just describe the content. "Revenue 2024" is not a message. "Revenue grows despite inflation" is.

Exploratory Data Analysis (EDA)

Before you can formulate a message, you need to understand the data.

Data visualization is the key to data exploration.

EDA helps identify patterns, outliers, and relationships in the data.

EDA in R

Data Storytelling

Data alone rarely convinces.

A story makes data memorable.

Context → Tension → Resolution → Action

03

Tech Stack

JavaScript, D3.js & Svelte in practice

The modern stack for web dataviz: D3 for the math, Svelte for rendering.

D3.js

Data-Driven Documents

Not a chart tool — an ecosystem of modules.

Scales · Shapes · Layouts · Projections · Axes · Transitions …


// Data → DOM elements
svg.selectAll("rect")
  .data(dataset)
  .join("rect")
  .attr("x", d => xScale(d.category))
  .attr("height", d => height - yScale(d.value));
D3 has been the standard since 2011. Important: it's not a "build me a chart" tool. It translates data into visual elements. .data() binds arrays to DOM elements, .join() handles enter, update, and exit.

D3.js vs. Observable vs. Observable Plot

D3.js

Flexible, but more boilerplate.
Ideal for custom visualizations.

Observable

Interactive notebooks with D3 integration.
Perfect for prototyping & sharing.

Observable Plot

High-quality charts with minimal code.
Great for quick analysis & standard charts.

D3 + Svelte

The ideal duo

D3 handles:

Scales, Layouts, Shapes,
Projections, Calculations

Svelte handles:

Reactive rendering,
Animations, State, DOM

No virtual DOM — Svelte compiles to native JS → top performance.

Svelte's reactive system is a perfect fit: when data changes, visualizations update automatically. D3 calculates, Svelte renders. The NYT Graphics Team uses exactly this stack!

LayerChart

Svelte + D3 — declarative chart components


<Chart data={salesData} x="month" y="revenue">
  <Svg>
    <Axis placement="left" />
    <Axis placement="bottom" />
    <Bars />
    <Tooltip.Root>
      <Tooltip.Item label="Revenue" />
    </Tooltip.Root>
  </Svg>
</Chart>
LayerChart abstracts D3 into declarative Svelte components. Quickly productive, but with D3 power behind it.

Charts & Patterns

Hierarchies, Networks & Maps

What D3 can do beyond bar and line charts.

Rendering: SVG vs. Canvas vs. WebGL

SVG  →  < 1,000 elements  ·  DOM  ·  CSS  ·  Accessibility ✓

Canvas 2D  →  up to 10,000 elements  ·  pixel-based

WebGL  →  100,000+ elements  ·  GPU  ·  3D

For most dashboard charts: SVG. For large datasets: Canvas. For geodata / 3D: WebGL.

Hierarchies

Trees, nesting, part-to-whole

d3.hierarchy() + d3.treemap() / d3.pack()

Treemap – area represents size. Good for comparisons across multiple hierarchy levels.

Networks & Relationships

Nodes & Edges

d3.forceSimulation() — physics-based layout

Force-directed graph — nodes find their equilibrium positions through simulated physics.
Sankey diagram — shows flow and distribution across stages.
Hierarchical edge bundling — reveals relationships within a hierarchy without visual noise.

Maps

Geographic data in spatial context

d3.geoProjection() — 50+ projections

⚠️ Choose projection deliberately — Mercator distorts areas!

Choropleth Map — classic for election maps, population data, etc.
